# Phase 2: 数据收集增强 (Source Coverage)

**Status**: 设计中
**Created**: 2026-03-08
**Goal**: 让用户确信数据收集完整，消除"我漏掉了什么"的疑虑

---

## 问题定义

### 当前痛点
用户需要从多个平台手动收集数据：
- 支付宝账单
- 微信账单
- 云闪付账单
- 银行账单
- 现金支出

**问题**: 导入后无法确定是否"收集完整"。用户总是怀疑：
- "招行信用卡导入了吗？"
- "上个月的数据齐了吗？"
- "现金支出有没有漏记？"

### 解决思路
提供一个**月度数据源收集状态视图**，像考勤表一样清晰展示：
- 哪些来源已导入
- 哪些月份缺失数据
- 实际导入的交易数量

---

## 核心概念

### Source (数据源)
数据来源的导出文件，不是财务账户：

| Source ID | 名称 | 类型 | 说明 |
|-----------|------|------|------|
| `alipay` | 支付宝账单 | semi-auto | 需手动导出 |
| `wechat` | 微信账单 | semi-auto | 需手动导出 |
| `yunshanfu` | 云闪付账单 | semi-auto | 需手动导出 |
| `bank` | 银行账单 | semi-auto | 需手动导出 |
| `manual` | 手工录入 | manual | 现金等零星记录 |

**注意**: 一个 Source 可能包含多个 Account 的交易（如支付宝账单包含花呗、余额宝等）。

### Coverage (覆盖度)
特定 Source 在特定月份的数据收集状态：

| 状态 | 表示 | 含义 |
|------|------|------|
| 数字 | `145` | 该月已导入 N 条交易 |
| `NA` | - | 该月未导入数据 |
| `0` | `0` | 用户确认该月无数据 |

## 界面设计

### 数据源收集状态视图

```
┌────────────────────────────────────────────────────────────────┐
│  数据源收集状态                    [2025年 ▼]                   │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  来源 \ 月份   1月    2月    3月    4月    5月    6月 ...      │
│  ─────────────────────────────────────────────────────────     │
│  支付宝账单    145    132    89~    NA     NA     NA           │
│  微信账单      203    189    156    NA     NA     NA           │
│  云闪付账单     23     31     28     NA     NA     NA           │
│  银行账单       NA     45     NA     NA     NA     NA           │
│  手工录入        0     12      8     NA     NA     NA           │
│                                                                │
│  ─────────────────────────────────────────────────────────     │
│                                                                │
│  说明:  数字 = 已导入交易数    NA = 未导入    0 = 确认无数据     │
│        ~  = 本月进行中（当前月份）                              │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 交互行为

| 操作 | 结果 |
|------|------|
| 点击 **NA** 单元格 | 打开导入页面，预筛选该来源/月份 |
| 点击 **数字** 单元格 | 跳转到交易列表，筛选该来源/月份的交易 |
| 点击 **~** 单元格 | 提示"本月未结束，数据不完整" |
| 长按/右键 **NA** | 标记为 "0"（确认该月无数据） |

### 视觉提示

- **当前月份**: 单元格带边框高亮
- **进行中标记**: 数字后带 `~` (如 `89~`)
- **近期导入**: 24小时内导入的单元格带绿色圆点
- **缺失提醒**: 连续缺失 2+ 个月的来源名称后带 ⚠️ 图标

### Coverage Data (查询生成，无需新表)

从现有 `transactions` 表查询：

```sql
-- 获取某来源某月的交易数
SELECT COUNT(*) as count
FROM transactions
WHERE source = 'alipay'
  AND strftime('%Y-%m', date) = '2025-03';

-- 获取所有来源的月度统计
SELECT
  source,
  strftime('%Y-%m', date) as month,
  COUNT(*) as transaction_count
FROM transactions
WHERE date >= '2025-01-01'
GROUP BY source, month;
```

### Optional: 显式标记无数据月份

如需支持 "0" 标记（确认该月无数据），需新增表：

```sql
CREATE TABLE source_coverage_zero (
  id TEXT PRIMARY KEY,
  source TEXT NOT NULL,
  year_month TEXT NOT NULL,  -- e.g., '2025-03'
  marked_at TEXT NOT NULL,
  UNIQUE(source, year_month)
);
```

## 实现考虑

### 性能
- 数据量少（月度聚合），无需优化
- 查询覆盖最近 12-24 个月即可

### 扩展性
- 新增 source 只需修改配置，无需 DB 变更
- 未来可支持自定义 source（用户自定义来源）

### 向后兼容
- 使用现有 `source` 字段，无需数据迁移
- 历史交易自动显示在对应月份

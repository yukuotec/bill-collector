## 核心功能

记账小助手是一个本地优先的桌面应用程序，旨在帮助用户统一管理来自多个支付平台（支付宝、微信支付、云闪付、银行账单）的个人支出，并提供智能去重、自动分类、家庭成员分配等高级功能。

### 主要特性

- **多源数据导入**: 支持支付宝、微信、云闪付、银行账单的CSV/XLSX/PDF格式
- **智能去重**: 自动检测并处理跨平台重复交易
- **自动分类**: 基于关键词的智能分类系统
- **家庭成员管理**: 支持多人家庭账本，可分配交易给不同成员
- **智能分配**: 基于历史行为的机器学习分配建议
- **仪表盘分析**: 丰富的可视化报表和钻取分析
- **预算管理**: 预算设置和超支提醒
- **邮件自动捕获**: 自动从邮箱下载账单附件
- **命令行工具**: 完整的CLI支持批量操作
- **本地优先**: 所有数据存储在本地SQLite数据库，无云同步

## 数据导入

### 支持的格式

| 来源 | 格式 | 说明 |
|------|------|------|
| 支付宝 | CSV, XLSX, PDF | 支持标准支付宝账单格式 |
| 微信支付 | CSV, XLSX, PDF | 支持标准微信支付账单格式 |
| 云闪付 | CSV, XLSX, PDF | 支持标准云闪付账单格式 |
| 银行账单 | CSV, XLSX, PDF | 支持通用银行账单格式 |
| 图片收据 | PNG, JPG | 支持OCR识别（需Tesseract.js）|

### 导入方式

1. **手动导入**: 通过文件选择对话框导入
2. **拖拽导入**: 直接拖拽文件到应用窗口
3. **邮件自动捕获**: 配置邮箱账户自动下载账单
4. **命令行导入**: 使用CLI工具批量导入

### 预览与验证

- 导入前显示预览（前5条记录）
- 自动检测文件编码（UTF-8/GB18030）
- 列映射自动识别（日期、金额、类型、商户等）
- 干运行模式（dry-run）验证解析结果

## 智能去重

### 去重策略

#### 1. 精确匹配 (Exact Match)
- **条件**: 同一天 + 相同金额 + 相同商户/描述
- **处理**: 自动合并，不保留重复记录

#### 2. 同期模糊匹配 (Same Period Fuzzy Match)
- **条件**: ±1天内 + 相同金额 + 相似描述（Levenshtein距离≤3）
- **处理**: 标记为待确认，用户可选择保留或合并

#### 3. 跨平台模糊匹配 (Cross Platform Fuzzy Match)
- **条件**: ±3天内 + 相同金额 + 相似商户名称
- **处理**: 标记为待确认，用户可选择保留或合并

### 退款检测

- 自动识别退款交易（包含"退款"、"退回"等关键词）
- 智能匹配原始交易（基于金额、商户、时间范围）
- 建立退款-原始交易关联

### 重复记录管理

- 专门的"重复记录"面板查看所有待确认重复项
- 一键操作：保留两条 / 合并
- 合并后保留原始交易，删除重复交易

## 家庭成员管理

### 成员管理

- 添加/编辑/删除家庭成员
- 为每个成员分配颜色标识
- 成员列表在交易分配时可用

### 交易分配

- 为每笔交易分配给特定成员
- 支持取消分配（设为未分配）
- 成员支出统计报表

### 智能分配（Phase 1）

#### 规则引擎
系统内置默认分配规则：

| 成员 | 关键词 |
|------|--------|
| 老公 | 游戏、数码、电子、汽车、烟、酒 |
| 老婆 | 化妆品、护肤、包包、服饰、美甲 |
| 孩子 | 学校、培训、玩具、奶粉、童装 |
| 家庭 | 水电煤、物业、买菜、日用品 |

#### 批量分配提示
- 当为某笔交易分配成员时，系统检查相似交易
- 如果发现2笔或更多相似交易已分配给同一成员，显示批量分配提示
- 用户可选择仅分配当前交易，或批量应用到所有相似交易

### 智能分配（Phase 2 - 机器学习）

- 基于历史分配行为学习模式
- 提取特征：商户、分类、描述、关键词
- 预测最可能的成员分配
- 置信度分级：自动分配（≥70%）、建议分配（≥30%）、无建议（<30%）

## 仪表盘分析

### 概览卡片

- 年度支出/收入/净额
- 当月支出/收入
- 预算提醒（超支/接近预算）

### 可视化图表

#### 月度趋势图
- 最近12个月支出/收入趋势
- 环比变化百分比
- 本月标识

#### 分类占比图
- 年度支出分类饼图
- 点击分类可钻取到交易详情
- 分类图标可视化

#### 成员支出统计
- 各成员年度支出柱状图
- 按成员颜色区分

#### Top商家表格
- 年度消费频次最高的商家
- 点击商家可钻取到交易详情

### 钻取分析

- **分类钻取**: 点击饼图分类 → 跳转到交易页面，按分类和年份过滤
- **商家钻取**: 点击Top商家 → 跳转到交易页面，按商家和年份过滤
- **URL参数**: 使用URL查询参数传递过滤条件
- **上下文芯片**: 在交易页面显示可移除的钻取条件芯片
- **清除功能**: 一键清除所有钻取条件

## 预算管理

### 预算设置

- 按月设置预算
- 支持分类预算（如餐饮预算）或总体预算
- 预算历史记录

### 预算提醒

- **正常**: 预算使用率 < 80%
- **警告**: 预算使用率 ≥ 80% 且 < 100%
- **超支**: 预算使用率 ≥ 100%

### 预算面板

- 在仪表盘显示当前月份的预算状态
- 进度条可视化预算使用情况
- 剩余金额/超支金额显示

## 命令行工具

完整的CLI工具支持以下命令：

### 导入命令
```bash
expense-cli import <file> --source alipay|wechat|yunshanfu|bank
expense-cli import --watch-dir <directory>  # 监听目录自动导入
```

### 查询命令
```bash
expense-cli list --category --source --month --duplicate-type --refund-only --limit
```

### 导出命令
```bash
expense-cli export --csv|--excel --output <file> --start-date --end-date
```

### 汇总命令
```bash
expense-cli summary --year --month
expense-cli recurring --min-occurrences 3  # 识别周期性支出
```

### 备份命令
```bash
expense-cli backup --output <file>                    # 本地备份
expense-cli backup --target s3 --s3-uri s3://bucket/path  # S3备份
```

### 配置命令
```bash
expense-cli config set watch_dir <directory>  # 设置监听目录
expense-cli config get watch_dir             # 获取配置
expense-cli config list                      # 列出所有配置
```

### 邮件命令
```bash
expense-cli email-import --source alipay,wechat --dry-run  # 邮件导入（需himalaya CLI）
```

## 数据管理

### 数据库

- **SQLite**: 本地嵌入式数据库
- **位置**: `~/Library/Application Support/expense-tracker/expenses.db` (macOS)
- **备份**: 手动导出数据库文件
- **S3备份**: 支持AWS S3云备份

### 导出功能

- **CSV导出**: 标准CSV格式，支持UTF-8 with BOM
- **Excel导出**: .xlsx格式，包含完整交易数据
- **选择导出**: 可选择特定交易导出

### 搜索与筛选

- **全文搜索**: 描述、备注、商户字段
- **高级筛选**: 日期范围、分类、来源、类型、成员、重复类型、退款
- **排序**: 按日期、金额排序（升序/降序）
- **分页**: 支持20/30/50条每页

### 标签系统

- 为交易添加自定义标签
- 标签管理（添加/删除）
- 标签搜索和筛选

## 技术架构

### 前端 (Renderer Process)
- **框架**: React 18 + TypeScript
- **状态管理**: React Hooks + URL Hash State
- **图表**: Recharts
- **样式**: CSS Modules

### 后端 (Main Process)
- **框架**: Electron + TypeScript
- **数据库**: SQLite via sql.js
- **CSV解析**: PapaParse
- **PDF解析**: pdf-parse
- **OCR**: Tesseract.js
- **邮件**: IMAP + mailparser

### 构建工具
- **打包**: Vite + electron-builder
- **测试**: Node.js built-in test runner
- **类型检查**: TypeScript
